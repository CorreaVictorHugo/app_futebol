Etapa 1: definir o projeto com clareza

Antes de escrever código, precisamos definir o que o sistema vai fazer na versão 1.

Objetivo da V1

Criar um app web para:

cadastrar jogadores
criar partidas
vincular jogadores a uma partida
registrar desempenho de cada jogador na partida
consultar histórico e métricas básicas
Funcionalidades da V1

1. Cadastro de jogadores

Cada jogador terá:

id
nome
apelido
posição
data de cadastro

2. Cadastro de partidas

Cada partida terá:

id
data
local
observação opcional
data de criação

3. Participação do jogador na partida

Essa é a parte mais importante do projeto.

Um jogador pode participar de várias partidas, e uma partida pode ter vários jogadores.
Então precisamos de uma tabela intermediária.

Essa tabela vai guardar:

id
jogador_id
partida_id
time
gols
assistencias
nota
presenca

Essa tabela é a que vai dar força para o projeto na área de dados.

Estrutura lógica do banco

Ficaria assim:

Tabela jogadores

Guarda os dados do jogador.

Tabela partidas

Guarda os dados da partida.

Tabela participacoes_partida

Liga jogador e partida, e guarda os números daquela atuação.
